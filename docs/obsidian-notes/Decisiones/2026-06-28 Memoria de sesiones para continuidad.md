# 2026-06-28 Memoria de sesiones para continuidad

## Estado

Aceptada.

## Decisión

Cada proyecto mantiene `Sesiones/En curso.md` durante el trabajo activo. Esta nota
resume acuerdos, cambios y pendientes relevantes sin copiar la conversación completa.

Al cerrar una sesión confirmada, la nota se convierte en
`Sesiones/<Fecha> - <Versión>.md` y debe incluir:

- reto o compromiso;
- resumen ejecutivo;
- tiempo y horas;
- acuerdos y decisiones;
- trabajo realizado;
- validaciones y evidencia;
- estado al cerrar;
- pendientes y continuidad;
- conexiones con proyecto, reglas, herramientas y decisiones.

`PROJECT_CONTEXT.md` enlaza la última sesión cerrada y `En curso.md` cuando exista. Las
notas históricas no se reescriben para fingir conocimiento posterior; pueden recibir
una nota de actualización que enlace decisiones que las superaron.
