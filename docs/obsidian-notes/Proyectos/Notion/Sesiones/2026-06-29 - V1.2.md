# Sesión 2026-06-29 — V1.2

> Estado: cerrada y confirmada por el usuario.

## Reto o compromiso

Consolidar una continuidad fiable para cualquier IA: descripción ejecutiva clara,
reglas globales y por proyecto, flujo obligatorio de inicio, memoria de sesión
suficiente, prompts coherentes y cierre controlado por el usuario.

## Resumen ejecutivo

La sesión refinó la arquitectura documental del proyecto para que Obsidian sea la
memoria suficiente de Codex y Claude Code, mientras Notion permanece como ledger de
métricas y capa de API. Se separaron reglas globales de reglas locales, se redujo
`PROJECT_CONTEXT.md` a un índice y se estableció una nota `En curso.md` para evitar
pérdida de contexto antes del cierre.

También se formalizó que la IA no decide cuándo termina una sesión: espera la indicación
del usuario, presenta el borrador y solicita confirmación antes de registrar en Notion.
Cada cierre confirmado conserva evidencia mediante commit y tag Git anotado e
inmutable.

## Acuerdos y decisiones

- [[Obsidian]] es la memoria narrativa y documental para la IA.
- [[Notion]] conserva sesiones, actividades, métricas, reportes y acceso por API; no se
  consulta para reconstruir contexto rutinario.
- [[Graphify]] conserva relaciones técnicas derivadas y se consulta antes del código.
- El flujo es `AGENTS/CLAUDE → PROJECT_CONTEXT → reglas → resumen → última sesión →
  En curso → Graphify → código`.
- `PROJECT_CONTEXT.md` es únicamente un índice mínimo.
- [[Reglas globales del Sistema de Trazabilidad]] aplica a todos los proyectos;
  `Proyectos/<Proyecto>/Reglas.md` contiene solo convenciones locales.
- Cada proyecto usa `Resumen.md`, `Reglas.md`, `Estado actual.md`, `Sesiones/`,
  `Decisiones/` y `Arquitectura/`.
- La IA espera a que el usuario indique cuándo cargar la sesión en Notion y solicita
  confirmación final después de mostrar el borrador.
- Cada versión cerrada usa un tag Git anotado e inmutable. `V1.1` no se modificó y
  esta sesión se publica como `V1.2`.
- Cada nota cerrada debe contener resumen, acuerdos, trabajo, evidencia, estado,
  pendientes, continuidad y conexiones.

## Trabajo realizado

- Se hizo una prueba real de reanudación desde V1.1 y se identificaron brechas.
- Se añadió una descripción ejecutiva consolidada al resumen del proyecto.
- Se separaron y documentaron reglas globales y reglas específicas de Notion.
- Se ajustaron `AGENTS.md`, `CLAUDE.md` y `PROJECT_CONTEXT.md`.
- Se actualizaron los prompts para proyectos nuevos y existentes en Obsidian y Notion.
- Se formalizó la autorización humana antes del registro de sesiones.
- Se completó la evidencia de V1.1 con resumen, commit y enlace de Notion.
- Se creó una plantilla estándar y una política de sesión en curso.

## Estado al cerrar

El proyecto dispone de una descripción ejecutiva consolidada, reglas globales y
locales, un índice mínimo portable, memoria para sesiones abiertas, notas cerradas
autosuficientes y prompts que reproducen la arquitectura en proyectos nuevos o
migrados. Notion permanece como ledger y no como memoria narrativa de la IA.
